# COLOR COMBO BOUND
https://github.com/user-attachments/assets/705ad965-19de-4105-919d-227cf295631a

本作は、物理演算を利用した2Dパズルアクションゲームです。
ゲームプログラマーとしての就職活動に向け、ゲーム開発未経験から個人で約1ヶ月間でUnityおよびC#を独学で習得しながら企画から実装までを行いました。

現在、以下のURL（itch.io）にてプレイ可能です。
[https://ono-mao.itch.io/color-combo-bound]

## 1. 概要とゲームシステム
決められたターン数（ショット回数）内で、画面内の敵をすべて倒すとクリアとなる2Dパズルゲームです。

単にボールを弾いて軌道を見守るだけでなく、プレイヤー自身が「マレット」を操作してボールを直接打ち返すことが出来ます。ボールを落とさずに連続して打ち返すことでコンボが加算され、敵に与えるダメージが増加するシステムを導入し、パズルゲームにアクション性を加えました。プレイヤーが自ら操作を行い、ゲームに深く関与しているという実感が持てることを意識して設計しています。

**主なギミック**
* **マレットスイッチ**: ボールが接触するとマレットが出現し、右クリック時のマウスのX軸移動に合わせてプレイヤーがボールを打ち返せるようになります。
* **拡散スイッチ**: ボールの速度を上昇させ、ランダムな角度へ反射させることで予測不能な軌道を生み、ゲームの緊張感を高めます。
* **レーザースイッチ**: 接触時に上下にレーザーを発射し、レーザーが当たった敵にダメージを与えます。

## 2. 操作方法
* **左クリック時ドラッグ**: ボールのショット予測線の出現
* **左クリックを離す**: ショット予測線の方向にボールをショット
* **右クリック時ドラッグ**: マレット出現時のマレットの左右移動
* **左クリック**: 各種UIの操作

## 4. 技術的なアピールポイント

本ゲームの作成では、将来的な機能拡張や、チーム開発等で他者がスクリプトを読むことを見据え、「保守性と可読性の高いコード設計」を意識して開発を行いました。

### 4.1. イベントシステムによる疎結合な設計
各クラス（Manager）が直接他のクラスを参照する状態を避けるため、Observerパターンを活用したイベント駆動型の設計を採用しています。
具体的には、`GameEvents.cs` に `System.Action` デリゲートを集約し、すべてのイベント（ターン変更、ボールの衝突、ギミックの発動、敵の撃破など）をこのクラス経由で通知し、各クラスが自身の必要な処理だけを実行する仕組みを構築しました。
これによりクラス間の結合度が大幅に下がり、コードの可読性向上や、UIの追加、新しいギミックの実装が容易になっています。

### 4.2. マジックナンバーとハードコードの排除
コード内の可読性を低下させるマジックナンバーや直接的な文字列の指定を排除しています。
* `TagName.cs` や `SceneName.cs` といった静的クラスを作成し、タグやシーン名を定数として一元管理することで、タイプミスによるバグを未然に防いでいます。
* 各Managerクラス内のステータス制限値や角度の限界値などは変数として明示的に定義し、計算ロジックの意味を読み取りやすくしています。

### 4.3. 物理演算とRaycastを活用した動的制御
* **MalletManager.cs**: マレットの可動域を制限する際、固定の数値を指定するのではなく、`Physics2D.Raycast` を用いて左右の壁（フレーム）までの距離を動的に計算し、ステージの形状に依存しない汎用的な制御を行っています。
* **LaserLauncherManager.cs**: レーザーの描画および当たり判定処理において、Raycastを用いて障害物までの距離を取得し、`LineRenderer` の描画終点をローカル座標系に変換して正確に反映させています。

## 5. テストプレイ時の要望と改善

初めに基本的なゲームシステムが完成した時点で、大学の教授や知人にテストプレイを依頼し、バグの有無や操作のわかりやすさについて確認を行いました。
結果として、致命的なバグはなく各種ギミックも効果的に機能していると好評を得た一方で、「マレットでボールを打ち返すのがこのゲームの主要な面白さであるため、その操作への導線がもっとあった方が良い」というフィードバックを受けました。

この意見を反映し、文字での説明画面だけでなく、実際のプレイ画面上で直感的に操作方法が理解できる「アニメーションでのチュートリアル」を最初のステージに新たに実装し、プレイヤーが迷わずにアクションを楽しめるよう改善を行いました。
